## 1.5 사용자 체감 변화

이제 브라우저 자동화 쪽은 단순 스캐폴드에서 한 단계 나아가, 사용자가 `chrome-debug.bat`으로 별도 Chrome을 열면 런타임이 CDP로 페이지 열기, 제목 읽기, 본문 텍스트 추출, 스크린샷 저장을 실행할 수 있는 기반이 생겼다. 아직 실제 Chrome 실측 전이라 자동화 가능 상태로 과장하지 않고, Chrome이 꺼져 있으면 바로 `chrome-debug.bat을 먼저 실행하세요`라는 안내를 받는다.

## 2. 변경 파일과 이유 (파일별 1~2줄, 신규/수정 구분)

- 신규: `workspace-template/agent_ops/adapters/browser_cdp.py`
  - `urllib` + 기존 `ws_min.WsClient`로 9222 CDP 탭에 연결하고 4개 browser action을 구현했다.
  - Chrome/CDP 오류는 예외로 올리지 않고 `{"ok": False, "action", "data", "error"}` 형태로 반환한다.
- 수정: `workspace-template/agent_ops/adapters/__init__.py`
  - browser adapter에 `execute: browser_cdp.execute`를 연결했다.
  - `available=False`와 pending 문구는 그대로 유지했다.
- 신규: `workspace-template/launch/chrome-debug.bat`
  - UTF-8/`py -3.11` guard 규약을 따르고, 별도 임시 프로필로 Chrome CDP 포트를 연다.
- 신규: `workspace-template/tests/test_browser_adapter.py`
  - Chrome 미기동 안내, action 라우팅, 잘못된 action 거부, BAT의 CDP 포트/별도 프로필 사용을 검증한다.
  - 9222가 열려 있으면 example.com live checks까지 실행하고, 없으면 SKIP exit 0 경로를 탄다.
