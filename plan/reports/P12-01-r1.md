## 1.5 사용자 체감 변화

아직 화면에서 바로 보이는 기능은 없지만, 다음 단계에서 Chrome을 제어할 때 `websocket-client` 같은 외부 패키지를 설치하지 않아도 된다. 회사/오프라인 PC에서 의존성 반입 부담이 줄고, 브라우저 자동화 기능을 더 쉽게 붙일 수 있는 기반이 생겼다.

## 2. 변경 파일과 이유 (파일별 1~2줄, 신규/수정 구분)

- 신규: `workspace-template/agent_ops/adapters/ws_min.py`
  - `ws://127.0.0.1` 전용 RFC6455 최소 client를 stdlib만으로 구현했다.
  - JSON text frame 송수신, client masking, ping 자동 pong, close, timeout, bad upgrade response 검증을 포함했다.
- 신규: `workspace-template/tests/test_ws_min.py`
  - 테스트 내부에 stdlib socket mini server를 구현해 외부 네트워크/패키지 없이 loopback 검증을 수행한다.
  - 짧은 payload, 126-length payload, 127-length payload, 한글 payload, ping/pong, close, timeout, invalid accept, binary/fragment rejection을 검증한다.

## 5. 계획과 다르게 한 것 (deviation — 없으면 "없음". 사소한 것도 전부)

- 테스트가 task의 예상 규모보다 커졌다. WebSocket frame 길이/마스킹/ping/close/bad accept를 외부 서버 없이 안정적으로 검증하려고 mini server helper를 테스트 파일에 포함했다.
