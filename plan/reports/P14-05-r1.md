# P14-05 r1 — 주간보고 초안(weekly_report)

## 1. 요약
- `weekly_report` capability를 등록하고 `agentops.py weekly` subcommand를 추가했습니다.
- 명령 실행 시 `agent_ops/results/reports/weekly_<YYYYMMDD>.md`를 생성합니다.
- 보고서는 지난 7일의 audit log, 완료 일정, 다음 주 예정, 생성 산출물을 기록 기반으로만 집계합니다.

## 1.5 사용자 체감 변화
- 사용자는 매주 빈 문서에서 시작하지 않고, 실제 기록 기반의 주간보고 초안을 바로 열어 보완할 수 있습니다.
- 초안 말미에 `TODO: 정성 성과/이슈는 직접 보완`을 남겨 자동 생성물이 완성 보고서처럼 보이지 않게 했습니다.

## 2. 변경 파일
- `workspace-template/agent_ops/capabilities.py`
  - `weekly_report` capability 추가
  - keywords: `주간보고`, `주간 보고`, `위클리`, `weekly`
  - artifact kind는 task 지시대로 전용 kind 없이 `document` 재사용
- `workspace-template/agent_ops/agentops.py`
  - `weekly` subcommand 추가
- `workspace-template/agent_ops/secretary.py`
  - audit/schedule/artifacts 기반 주간보고 생성 로직 추가
  - 빈 섹션은 `없음`으로 출력
  - 자료 원천 경로를 report 본문에 명시
- `workspace-template/tests/test_secretary.py`
  - fixture 기반 weekly E2E와 empty fixture checks 추가
- `workspace-template/tests/test_capability_bench.py`
  - `weekly_report` routing + `document` artifact plan 회귀 추가

## 4. DoD 확인
- 3개 원천 반영:
  - audit: 지난 7일 kind별 건수와 대표 task 문구
  - schedule: 지난 7일 done 일정과 다음 주 open 일정
  - artifacts: 지난 7일 생성/수정 파일 수와 대표 경로
- 초안 명시:
  - `Weekly Report Draft`
  - `TODO: 정성 성과/이슈는 직접 보완`
- 기존 checks:
  - secretary/capability bench/full regression 모두 통과

## 5. 제한 및 리뷰 포인트
- schedule store에는 별도 완료 시각 필드가 없으므로, 완료 일정의 "지난 7일" 판정은 현재 저장된 due date 기준입니다.
- artifact 기간 판정은 파일 mtime 기준입니다.
- audit에 없는 활동은 추정하지 않고 기록된 task 문구만 대표 항목으로 출력합니다.
