## 1. 결과 요약 (상태 어휘)

- resolved: **GGUF/ASR 모델 3파일의 공식 SHA256을 HF LFS pointer로 확정**(수 GB 다운로드

## 5. 계획과 다르게 한 것 (deviation)

- 구현자가 Codex가 아닌 Fable(사용자 지시).
- 매니페스트를 별도 파일이 아니라 dependencies.json의 `prefetch_files`에 통합(단일 소스).
- 모델 해시는 "다운로드→certutil" 대신 **HF LFS pointer의 공식 sha256**을 사용(수 GB 회피,
  값은 동일한 공식 published 해시). wheel은 실제 다운로드→해시.

## 6. 하지 않은 것 / pending

- **home-PC pending**: GitHub 릴리스 바이너리 3종(llama.cpp win zip / whisper.cpp win /
  ffmpeg win64)의 다운로드+SHA256. 이 클라우드 환경 proxy가 GitHub 릴리스를 차단하므로
  **codex-in-cloud도 동일하게 불가** — 집/개발 PC 또는 사용자 반입 단계에서 확정.
  확정 후 dependencies.json의 해당 status를 resolved로, sha256을 실값으로 갱신하면
  verify_prefetch/test가 그대로 검증한다.
- 모델 wheel과 달리 GGUF/ggml 파일 자체 다운로드는 하지 않음(공식 해시만 확정) — 반입
  시 다운로드 후 verify_prefetch로 대조.
