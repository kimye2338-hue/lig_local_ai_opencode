## 필수 수정 (잔여 1건)

1. `tests/test_batch_adapters.py`의 **exit-53 fake exe(POSIX unix_body)** — 문제:
   UTF-16LE 바이트를 `printf '\xff\xfeE\x00…'`로 emit하는데, **리눅스 `/bin/sh`(dash)는
   `\xNN`을 해석하지 않고 리터럴 4바이트(`\`,`x`,`f`,`f`)로 출력**한다. 그래서 log_tail이
   깨지고 `check("autocad exit 53 classified …", … and "ErrorStatus=53" in exit53["log_tail"])`
   가 실패 → **파일이 여전히 POSIX에서 RED**(이후 check도 실행 안 됨). 리뷰어 실측:
   - `printf '\xff…'` on dash → `od`: `5c 78 66 66 …`(리터럴 `\xff`), 디코드 깨짐.
   - `FAIL autocad exit 53 … log_tail: '硜晦硜敦…'`. (어댑터는 정상 — `ok:False`,
     `"exit 53" in error`, `returncode:53`. 순수 테스트 바이트 emit 문제.)
   / 요구: dash가 지원하는 **octal escape**로 바꿔라(아래 "되는 방법"). 또는 log_tail 내용
   assertion만 `os.name=="nt"`로 가드하고 classification(returncode/에러)만 크로스플랫폼으로
   남겨라(UTF-16 stdout assertion과 동일 패턴).

## 되는 방법 (리뷰어 검증 완료 — 둘 중 택1)

**(권장) octal escape** — dash `printf`는 `\NNN`(8진)을 실제 바이트로 낸다. exit-53
unix_body의 `\xff\xfeE\x00r\x00…`를 아래로 교체:

```
\377\376E\000r\000r\000o\000r\000S\000t\000a\000t\000u\000s\000=\0005\0003\000
```

리뷰어 실측: 이 octal 문자열 → `od`: `ff fe 45 00 72 00 …`(실 UTF-16LE) →
`.decode("utf-16-le")` == `"﻿ErrorStatus=53"` → `"ErrorStatus=53" in log_tail` 통과.
(Python 소스 문자열에선 `\\377\\376E\\000…`로 이스케이프해 .sh에 `\377…`이 쓰이게 할 것.)

**(대안) log 내용만 nt 가드** — classification은 전 플랫폼, log_tail 내용만 Windows:

```python
ok53 = exit53["ok"] is False and "exit 53" in exit53["error"] and exit53["returncode"] == 53
if os.name == "nt":
    ok53 = ok53 and "ErrorStatus=53" in exit53["log_tail"]
check("autocad exit 53 classified as dwg open problem", ok53, exit53)
```

리뷰어 실측: 대안의 크로스플랫폼 부분(`ok False`+`exit 53`+`returncode 53`)은 POSIX
실행에서 그대로 참(위 어댑터 반환값).
