## 권고 (선택 반영 — 채택 여부를 다음 보고서에 기록할 것)

- `_open_copy`에서 사본 생성(shutil.copy2) 후 Excel 기동이 실패하면 방금 만든 사본
  파일이 디스크에 남는다 — 원본 불가침에는 해가 없으나, 실패 분기에서 그 사본을
  삭제(unlink)하면 잔류물이 없다.
- `xl.Run(macro_name)`은 매크로가 무한 루프면 영구 대기한다 — P15-04 실측 시 타임아웃/
  감시 필요 여부를 확인 항목으로 남길 것 (지금 코드 수정 불요).

## 확인된 것 (잘한 것 — 다음 작업에서도 유지)

- 사본 강제가 API 표면 수준에서 성립: 공개 action 6종에 원본 열기 경로 자체가 없고,
  `open_copy`는 `사본_<이름>`(충돌 시 `_2` 증가) 사본만 연다. SaveAs 부재로 원본
  덮어쓰기 경로 없음 — task 금지 조항 준수.
- COM 시퀀스가 실증 순서(DispatchEx → Visible=False → DisplayAlerts=False →
  Workbooks.Open(사본) → VBComponents.Add(1) → AddFromString → Run → Close(False) →
  Quit → CoUninitialize) 그대로. VBProject 예외는 `fallback: manual_import` +
  Alt+F11 guide로 강등, 예외 전파 없음.
- pywin32 optional import가 모듈 로드까지 안전 — pywin32 없는 이 리뷰 환경에서 import·
  전 API 호출 crash 0, 지정 안내 문구 반환.
- fake COM 세션으로 VBProject 차단 경로를 LLM/Excel 없이 검증한 테스트 설계.
- adapter 등록은 `execute` 연결만 하고 `available=False` 유지 — hard gate 준수.
