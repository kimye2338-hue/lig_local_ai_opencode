## 2. 최종검토에서 발견·수정한 결함 (7건 — 직접 구현분 포함 예외 없음)

1. **`docs/OFFLINE_REHEARSAL.md` §1 stale** — deferred 스코프 변경 이전 문구(3종 필수)가 잔존.
   → 파일럿 세트(wheel 8+embed)의 **구체 명령어**로 재작성. *교차파일 정합은 스코프 변경 커밋에서 같이 갔어야 함.*
2. **`docs/OPENCODE_INTEGRATION.md` provider 예시 불일치** — 예시 모델이 "local llama.cpp"로 게이트웨이 스토리와 모순, 라우트 형태도 실제(`/gateway/<model-route>/v1`, EXAONE-4.5-33B)와 다름. → `lig_providers.py` 기본값과 일치하게 수정.
3. **`verify_prefetch.py` deferred 침묵** — deferred 항목이 아무 줄도 안 찍힘(보고 누락). → `DEFER` 라인 추가.
4. **`verify_prefetch.py` 파일럿 스코프 미반영** — resolved인데 부재하면 전부 failure. 파일럿 불필요 모델(GGUF/ggml)을 안 받는 게 정상인데 exit 1. → 카테고리(`llm-gguf`/`asr-model`/`binary-github`)는 부재 시 `OPT`(존재 시엔 해시검증 유지). **실측: 파일럿 세트만으로 exit 0.**
5. **python-embed 해시 미확정 방치** — `PENDING_PREFETCH`였으나 python.org는 프록시 허용 → **실측 확정** `009d6bf7e3b2ddca3d784fa09f90fe54336d5b60f0e0f305c37f400bf83cfd3b` (11,249,023 bytes; zip 무결성 + python.exe/python311.dll 존재 확인). prefetch_files에 resolved 등재, setup.bat [STOP] 안내도 실경로로.
6. **opencode-binary 조달 경로 미반영** — `PENDING_PREFETCH`로 남아 있었으나 실제로는 CI(`build-offline-package.yml`)가 클론→패치→빌드→SHA256SUMS 내장→installer 자가검증까지 수행. → `SELF_VERIFIED_IN_ARTIFACT`로 정정. 구식 `python-wheels: not_needed` 항목도 prefetch 8종과의 표면 모순 해소(core=stdlib 불변 / adapters=prefetch_files).
7. **`test_release_manifest.py` 존재 브랜치 동일 결함** — 항상 SKIP이라 실행된 적 없던 브랜치를 **처음 실행해 보니** #4와 같은 결함(모델 부재=FAIL). → verify_prefetch와 동일 원칙(OPT)으로 수정. *한 번도 실행 안 된 브랜치는 검증 0 — SKIP 경로가 있는 테스트는 채운 상태 실측을 반드시 1회 확보할 것.*

## 5. 피드백 (다음 사이클용)

- **스코프 변경은 참조 그래프째로**: deferred 재분류 때 매니페스트+테스트는 갔지만 절차서(§1)가 남았다. 상태를 바꾸면 그 상태를 인용하는 모든 문서를 같은 커밋에서 grep으로 확인.
- **SKIP 가드가 있는 테스트는 "채운 상태" 실측을 1회 이상**: 존재 브랜치 결함(#7)은 브랜치가 실행된 적이 없어서 살아남았다.
