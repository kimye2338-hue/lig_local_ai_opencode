## 필수 수정

   문제: `ROOT`는 `core.ROOT`(=`AGENTOPS_ROOT` 데이터 루트, relocated install용)인데
   **RUNBOOK.md는 코드 디렉터리(`workspace-template/docs/`)에 shipped**된다. 그래서 코드≠
   데이터 루트(문서화된 relocated install)에서 doctor operations가 **`runbook: false`로
   오보고** — 장애 대응 중 운영자가 "RUNBOOK 없음"으로 오인한다. P15-03과 같은 CODE_ROOT
   vs 데이터 ROOT 계열(self-review 체크리스트 2). 리뷰어 실측:
   - 기본(ROOT=코드) → `runbook: True`
   - `AGENTOPS_ROOT=<데이터폴더>` → `runbook: false` (RUNBOOK.md는 실재)
   / 요구: shipped 문서이므로 **코드 루트** 기준으로 존재 검사하라(아래 "되는 방법").
   나머지 operations 필드(audit=LIG_AUDIT_DIR, reports=RESULTS, schedule=store)는 사용자
   데이터가 맞으므로 그대로 둔다.
