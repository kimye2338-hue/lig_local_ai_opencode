## 되는 방법 (리뷰어가 조각별로 검증)

**(a) fake 실행파일을 크로스플랫폼으로** — POSIX는 `.sh`+chmod, Windows는 `.cmd`:

```python
import os, stat
def _write_fake_exe(base: Path, *, win_body: str, unix_body: str) -> Path:
    if os.name == "nt":
        p = base.with_suffix(".cmd"); p.write_text(win_body, encoding="ascii")
    else:
        p = base.with_suffix(".sh"); p.write_text(unix_body, encoding="utf-8")
        p.chmod(p.stat().st_mode | stat.S_IXUSR | stat.S_IRUSR)
    return p
```

리뷰어 실측: `.sh`(`#!/bin/sh` + `exit 0`, chmod +x) fake exe로 `autocad_batch.execute`를
돌리면 **원본 DWG 불변·`/i 사본_*.dwg` 사용·copy exists**가 리눅스에서 그대로 통과했다.

**(b) UTF-16LE stdout 내용 assertion만 Windows 한정** — `"유니코드 로그" in stdout_tail`
같은 검사는 AutoCAD의 Windows 콘솔 출력 형식이라 POSIX 셸로 재현 불가:

```python
if os.name == "nt":
    check("autocad decodes utf16le stdout", "유니코드 로그" in ok_cad.get("stdout_tail",""), ok_cad)
```

이렇게 하면 부재 안내·사본 불변·exit 53 분류(mock)·**artifact-kind 라우팅/생성/품질·
registry** check가 전 플랫폼에서 실행된다. (리뷰어가 이 검사들을 리눅스에서 개별 실측해
전부 통과 확인 — 아래 "확인된 것".)

## 확인된 것 (리뷰어 실측 재현 — 어댑터/artifact 실체는 완전 정상)

- **matlab_batch**: find_matlab(env `MATLAB_EXE`→PATH→표준경로→""), script/exe 부재 시
  crash 없이 안내, `-batch "run('<name>')"` + cwd=script 폴더 + timeout 300 + audit.
  MATLAB Engine API 미도입(subprocess만) — 금지 준수.
- **autocad_batch**: 원본 DWG를 `사본_<이름>`으로 복사 후 `/i 사본 /s scr` 실행 —
  **원본 bytes 불변 실측 확인**. UTF-16LE 디코드(`text=True` 미사용), exit 53 분류,
  timeout, audit. accoreconsole 탐색(env override 포함).
- **autocad_script artifact kind (hard gate)**: 기존 `office_cad_automation` capability에
  kind만 추가(신규 capability/키워드 없음 → 오라우팅 위험 없음). 생성 `작업.scr`(.scr),
  quality ok, **사본 정책 문구·QSAVE 부재·pending 표기** 확인, **QSAVE 삽입 시 품질이
  실패로 잡음** 실측. `ARTIFACT_KIND_INFO`/`_KIND_SUFFIXES(.scr)` 등록 일관.
- **registry**: matlab/autocad 모두 `available=False` + app validation pending 유지.
