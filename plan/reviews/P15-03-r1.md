## 필수 수정

1. `agent_ops/adapters/outlook_com.py:_run_child` (146~156) — 문제: 자식 subprocess를
   `cwd=str(ROOT)`로 띄운다. 그런데 `core.ROOT`는 **데이터 루트**(`AGENTOPS_ROOT`,
   core.py:17 "relocated installs" 용)라 **코드 패키지가 없는 폴더**일 수 있다. 그 경우
   `python -m agent_ops.adapters.outlook_com`가 `agent_ops`를 import하지 못해 stdout이
   비고, 부모는 `"Outlook adapter returned no JSON"`을 반환한다 → **read_calendar/
   read_inbox/sync_calendar 전부 실패**. 리뷰어 실측:
   - `AGENTOPS_ROOT=<임시 데이터폴더>` 설정 후 `execute("read_calendar",{"days":7})`
     → `{'ok': False, 'error': 'Outlook adapter returned no JSON'}` (재현됨).
   - 반면 `execute`를 코드 루트에서 AGENTOPS_ROOT 없이 호출하면 정상
     (`pywin32 미설치` 안내). 즉 **코드≠데이터 루트 분리 시 깨지는 실사용 버그**다.
   - `tests/test_office_adapters.py`는 상단에서 `AGENTOPS_ROOT=tmp_root`로 이 조건을
     이미 만든다 → 리뷰 환경(패키지가 전역 import 불가)에서 이 테스트가 **RED**
     (`outlook absence path explains pywin32` 실패). 워커 Windows 환경은 agent_ops가
     ambient하게 import돼 우연히 초록이었을 뿐, 설계가 환경 의존적이다.
   / 요구: 자식이 AGENTOPS_ROOT와 무관하게 패키지를 import하도록 **코드 루트**를 명시하라
   (아래 "되는 방법" 채택). 수정 후 `test_office_adapters.py`가 코드≠데이터 환경에서도
   통과해야 한다(현 테스트가 그대로 검증 역할).

## 되는 방법 (리뷰어가 검증한 수정 — 그대로 채택 가능)

`_run_child`에서 cwd를 코드 루트로 바꾸고 PYTHONPATH도 함께 준다(cwd 재정의에도 견고):

```python
# 모듈 상단 상수
CODE_ROOT = Path(__file__).resolve().parents[2]   # = agent_ops 패키지의 부모(workspace-template)

# _run_child 내부
env = os.environ.copy()
env["PYTHONUTF8"] = "1"
env["PYTHONIOENCODING"] = "utf-8"
env["PYTHONPATH"] = str(CODE_ROOT) + os.pathsep + env.get("PYTHONPATH", "")
result = subprocess.run(cmd, cwd=str(CODE_ROOT), capture_output=True, timeout=timeout, env=env)
```

리뷰어 실측: `parents[2]` = `…/workspace-template`(정확). 임시 데이터폴더를 cwd로 두고
`PYTHONPATH=CODE_ROOT`(또는 `cwd=CODE_ROOT`)만 주면 자식이 정상 JSON을 출력
(`{"ok": false, "error": "pywin32 미설치 …"}`) → `execute`가 부재 안내를 되돌려준다.

## 피드백 (코칭 — 다음 작업의 습관으로)

- **좋았던 것**: 실측 hang 지침(GetActiveObject/서브프로세스 격리/UTF-8)을 코드에 정확히

  바로 그 조건이라 잡혔다).
