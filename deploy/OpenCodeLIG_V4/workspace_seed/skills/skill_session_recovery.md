# skill_session_recovery — 재시작 후 작업 재개

## 언제
PC 재부팅, OpenCode 재시작, 세션이 끊긴 뒤 사용자가 "계속"이라고 할 때.

## 절차
1. `checkpoints/CHECKPOINT_LATEST.md` read → `다음 행동` 확인.
2. `memory/MEMORY.md` read → 반복 실수 목록 상기.
3. 체크포인트의 `수정된 파일` 중 핵심 1~2개를 read 도구로 현재 상태 확인
   (체크포인트 이후 바뀌었을 수 있음).
4. `다음 행동`을 실행한다. 불명확한 점이 있으면 질문 1개만.
5. 재개 보고: "체크포인트(날짜)에서 재개, 지금 X를 합니다" 1~2줄.

## 체크포인트가 없으면
- 새 작업으로 간주하고 사용자에게 목표를 1문장으로 확인.
- 작업 시작 후 첫 의미 있는 결과가 나오면 즉시 CHECKPOINT_LATEST.md 생성.
