# skill_ui_automation — 프로그램 창 조작 (Phase C: 아직 미설치)

## 현재 상태
이 PC에는 **로컬 프로그램 창(UI) 자동 조작 모듈이 아직 설치되어 있지 않다.**
"이 프로그램에서 버튼 눌러줘 / 입력해줘" 요청을 받으면:
1. 솔직하게 안내한다: "화면 조작은 아직 준비 중입니다."
2. 대신 지금 가능한 것을 제안한다:
   - 파일 기반 작업(설정 파일 수정, 스크립트 생성, 배치 작업)은 지금도 가능.
   - 수행 절차를 단계별 체크리스트로 만들어 주는 것도 가능.
3. 절대 되는 척하지 않는다.

## 설치되면 따를 규칙 (Phase C 설계 — docs/AUTOMATION_ROADMAP.md 참고)
- 사용자가 보고 있는 화면에서, 지시받은 창/버튼만 조작한다.
- 각 동작 전 계획(어느 창, 어느 버튼, 어떤 입력)을 말하고, 위험하면 승인부터.
- 반복 작업은 1회 시연 → 확인 → 나머지 반복 순서로.
- 실패하면 즉시 멈추고 현재 화면 상태를 설명한다.
- 작업 후: 클릭/입력/변경 내역을 요약 보고, 체크포인트에 기록.
