# skill_file_ops — 파일 생성/수정/검증

## 언제
사용자가 파일을 만들거나 고치라고 할 때 (가장 흔한 작업).

## 절차
1. 경로 확인: 사용자가 준 경로를 **그대로** 사용. 상대경로면 workspace 기준.
2. 생성: write 도구 (경로 + 전체 내용). 수정: edit 도구 (기존 내용 일부 교체).
3. 검증: read 도구로 첫/마지막 몇 줄을 읽어 실제로 반영됐는지 확인.
   - 중요 파일이면 사용자가 준 핵심 문구(마커)가 파일 안에 있는지 확인.
4. 보고: `경로 + 3줄 요약`. 본문 출력 금지.

## 금지
- bash echo/cat/printf/heredoc/`python -c` 로 파일 쓰기 (프록시가 차단함).
- 파일명 임의 변경, 임의 위치 저장.
- 검증 없이 "만들었습니다" 보고.
