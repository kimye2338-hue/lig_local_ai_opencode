# skill_code_patch — 코드 수정/패치 워크플로

## 언제
"이 코드 고쳐줘", "버그 수정해줘", "이 함수 바꿔줘" 류의 요청.

## 절차
1. **읽기**: 대상 파일의 관련 부분만 read. 수정 범위를 먼저 한 줄로 말한다.
2. **백업**: 큰 변경(파일의 1/3 이상 교체, 구조 변경)이면 수정 전에
   `파일명.bak_YYYYMMDD` 로 사본을 write 도구로 만든다. 작은 수정은 생략.
3. **수정**: edit 도구로 필요한 부분만 바꾼다. 무관한 코드 정리/리포맷 금지.
4. **검증**: 가능한 가장 싼 방법으로 —
   - Python이면 `python -m py_compile "파일"` 실행.
   - 실행 가능한 테스트/명령이 있으면 1회 실행하고 결과 요약.
   - 아니면 read로 수정부만 재확인.
5. **보고**: 수정 파일 목록 + 무엇이 왜 바뀌었는지 2~3줄 + 검증 결과 1줄.

## 금지
- 요청 범위 밖 대규모 리팩터링.
- 검증 없이 "고쳤습니다" 보고.
- 실패한 테스트를 성공처럼 보고.
