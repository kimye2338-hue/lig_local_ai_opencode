# skill_risky_confirm — 위험 작업 확인 절차

## 언제
삭제/대량 이동/덮어쓰기/git push/설정 변경 등 되돌리기 어려운 작업 직전.

## 절차 (계획 먼저, 실행은 승인 후)
1. **계획 제시** (실행하지 말고):
   - 무엇을: 대상 경로/파일 개수
   - 왜: 한 줄 사유
   - 영향: 사라지거나 바뀌는 것
   - 복구: 백업 위치 또는 되돌리는 방법
2. "진행할까요? (예/아니오)" 로 **한 번만** 묻는다.
3. 승인 후 실행. 삭제 대신 가능하면 `OpenCodeLIG_USERDATA\backups\` 로 **이동**.
4. 결과 보고: 실제로 한 일 + 복구 경로.

## 판단 기준
- 파일 1~2개 생성/수정 = 위험 아님 → 묻지 말고 실행.
- 내용이 사라질 수 있음 / 여러 파일 / 시스템·설정 대상 = 위험 → 이 절차.
- 사용자가 이미 같은 세션에서 "다 지워도 돼"처럼 명시했으면 재확인 불필요.
