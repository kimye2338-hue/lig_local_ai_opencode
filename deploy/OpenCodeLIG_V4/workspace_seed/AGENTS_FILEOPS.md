# AGENTS_FILEOPS.md — 파일 작업 정책 (config instructions로 항상 로드됨)

- 파일 생성/수정은 **반드시 write 또는 edit 도구**로 한다.
- bash 도구의 echo / cat / printf / heredoc(`<<EOF`) / `python -c` 로 파일을
  만드는 것은 **금지**다. (프록시 가드가 차단하고 다시 요구한다.)
- 사용자가 준 **파일 경로를 정확히 그대로** 사용한다. 이름을 바꾸지 않는다.
- 쓰기 후 답변은 **파일 경로 + 3줄 요약**만. 파일 전체 본문을 채팅에 출력하지 않는다.
- 도구 호출로 파일을 만들 수 없는 상황이면, 만들 수 있는 척하는 bash 명령을
  출력하지 말고 한 문장으로 불가 사유를 말한다.
- 중요한 파일을 만들었으면 read 도구로 1회 읽어 핵심 내용(마커/첫 줄)이
  실제로 존재하는지 확인한 뒤 완료를 보고한다.
