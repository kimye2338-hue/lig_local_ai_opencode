# AGENTS_MEMORY.md — 지속 메모리 규칙 (필요할 때 읽는 참고 문서)

## 위치
- 지속 사실: `memory/MEMORY.md` (이 workspace 안 — 도구로 바로 읽고 쓸 수 있다)
- 작업 상태: `checkpoints/CHECKPOINT_LATEST.md`

## MEMORY.md에 기록할 것 (짧은 불릿, 날짜 포함)
- 확정된 환경 사실 (경로, 실행 파일 위치, 동작이 검증된 명령 문법).
- 반복된 실패와 그 원인 (예: cp949 디코드, 경로 따옴표, -q 미지원, 플러그인 지연).
- 사용자의 지속적 선호 (언어, 출력 형식).

## 기록하지 말 것
- API 키, 내부 URL 등 비밀 (이미 secrets 파일에 있다).
- 일회성 작업 내용 (체크포인트에 쓸 것).
- 로그 원문.

## 갱신 방법
- edit 도구로 해당 섹션에 불릿 추가/수정. 전체 재작성 금지.
- 같은 사실이 이미 있으면 중복 추가하지 않는다.
- 틀렸다고 판명된 항목은 지우지 말고 `~~취소선~~ (사유, 날짜)`로 남긴다.
