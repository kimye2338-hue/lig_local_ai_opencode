# 레퍼런스 지식베이스 — 작성 규약 (수집 에이전트/사람 공용)

이 폴더는 **큐레이션된 사실**의 층이다(개인 기억과 분리). 모델이 "이거 만들어줘"에
정확한 근거로 즉시 답하도록, 공학 도메인·규격·소프트웨어 API를 담는다.

## 노트의 역할 (중요 — 오해 방지)
노트는 **개념·공식·절차의 가이드**다. **구체 수치의 최종 근거가 아니다.**
- 표준 공식(σ=Mc/I, Miles 방정식, Taylor V·T^n=C 등)·유도·판정기준은 노트가 권위 있게 담는다(안정된 이론).
- **재료 물성(E·Sy·K_IC)·규격 수치(PSD g²/Hz·안전율·열처리 온도·절삭속도)는 노트가 최종값이 아니다** —
  반드시 원문(재료증명서/MMPDS/MIL-STD 원문/공구 카탈로그)에서 확인. 노트엔 "확인 필요"로 표기.
- 이유: 이 수치들은 로트·버전·조건별로 바뀌고, 방산에선 틀린 수치가 곧 안전/임무 실패다.

## 폴더
- `apis/` 소프트웨어 API·명령 — **레시피형**(정확한 호출 시그니처·버전 고정), 교과서 산문 아님.
  SolidWorks 2022 / AutoCAD 2019 / Fluent 2024R1 / MATLAB R2024a / Excel·Outlook VBA
- `domains/` 공학 도메인 — **교과서 챕터 수준**(공식·유도·워크드 예제·출처). 현재 11개 완성.
- `standards/` 규격 — 절차/구조 가이드, 수치는 원문 확인(MIL-STD-810H 등)
- `lifeskills/` 소프트스킬 — **문서작성·논문작성만** KB 대상.
  ※ 협상·커리어·ADHD·성과관리는 레퍼런스 사실이 아닌 코칭/개인 영역 → KB 제외(개인 기억으로).
- `_moc/<도메인>.md` Maps of Content — 도메인 허브(하위주제 지도)

## 노트 규약 (프론트매터·팩트체크)
```yaml
---
title: 진동해석
domain: 진동해석
aliases: [진동, vibration, 모달, 랜덤진동, miles, ...]   # 라우팅 키워드. 구체 기술어만(일반어 금지 — 오주입)
sources: [Rao "Mechanical Vibrations", Miles(1954), ...]  # 출처 없는 주장 금지
verified: true                    # 교차검증 통과(아니면 draft → 주입 시 '미검증' 꼬리표)
confidence: high                  # high | medium | low
numeric_claims: unverified        # 수치는 원문 확인 필요(주입 헤더에 ⚠️ 자동 노출)
version: (해당 시 SW/규격 버전)
reviewed: 2026-07-07
---
```
본문은 `## 주제` / `### 하위주제` 로 나눈다(주입기가 질문에 맞는 하위섹션만 골라 넣는다).
- 상단에 `> ⚠️ …` 한 줄 요지·경고(발췌 시 항상 살아남음). 신규 노트는 **"핵심 공식 요약" 블록**
  포함 권장(짧은 발췌로도 공식이 살아남게).
- **워크드 예제 2~3개**(약한 모델에 few-shot 역할 — 최고 레버리지).

## 출처 정책 (망분리 방산 — 배포 가능 여부를 가름)
- **공개 문헌만.** 표준 교과서(Shigley/Incropera/Rao/Fleeman 등)·공개 규격 요약·공개 API 문서.
- **사내/통제(CUI)/기밀 데이터를 KB에 넣지 말 것.** KB는 배포 패키지에 포함되므로 통제정보 유입 금지.
- 개인 기억(USERDATA)↔KB **보안경계**: USERDATA(기억/위키/일정)는 사용자 사적 데이터,
  KB는 배포되는 공개 레퍼런스. **USERDATA→KB로 자동 쓰기 경로를 만들지 말 것**(사적/통제정보 누출 방지).

## 신선도(staleness)
- `reviewed:` 날짜 유지. 버전 종속 정보(SW/규격)는 version 명시.
- 규격 개정(예 810H 새 판)·SW 버전 변경 시 해당 노트 재검토. 오래된 version은 환각 유발.

## 신규 노트 완료 정의 (Definition of Done)
1. 프론트매터 완비(verified/confidence/numeric_claims/sources).
2. **라우팅 골든셋 케이스 동반**(`tests/test_knowledge_routing.py`): positive(이 노트가 뽑혀야 할 질문)
   + **negative(뽑히면 안 되는 일상어 질문)**. 허브/캡스톤 노트일수록 트리거어가 일상어와 겹쳐 오주입 위험 큼.
3. 교차링크([[다른노트]]) 연결. 24/24 회귀 통과 확인.

## 검색 동작
작업 프롬프트 → 희소용어 가중 라우팅(변별력 있는 의도어 우선) → 도메인 MOC(지도) + 노트의
질문 관련 하위섹션 주입(헤더에 출처·신뢰도·수치경고). 무관 프롬프트엔 주입 0(오주입 방지).
`python -m agent_ops.knowledge_base "진동 시험 지그"` 로 무엇이 주입되는지 확인.
