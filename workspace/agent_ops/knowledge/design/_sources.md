# 디자인 가이드 출처

`document_slides.md`의 원칙 근거. 정립된 공개 원칙 기반이며, 특정 저작물 문구를
그대로 복제하지 않고 원칙만 요약했다(라이선스 안전).

- Robin Williams, *The Non-Designer's Design Book* — CRAP: Contrast/Repetition/Alignment/Proximity.
- Edward Tufte, *The Visual Display of Quantitative Information* — data-ink ratio, chartjunk 제거.
- 프레젠테이션 통념 — 1 슬라이드 = 1 메시지, 제목은 주장, 글머리 최소화.
- Effective design doc 구조 — 배경/문제 → 목표·비목표 → 제안 → 대안·트레이드오프 → 리스크 → 결정.

사용자 제시 자료(2026-07-06) 검토 반영분:
- refactoringenglish.com "Write an Effective Design Doc" — 문서 구조/작문 규칙.
- getdesign.md, styles.refero.design, awesome-design-skills — 전이 가능한 디자인 원칙만 추출
  (웹 전용 요소는 제외). 상세 검토: docs/EXTERNAL_TOOLS_REVIEW.md.
