# 열유체해석(CFD) 기초

## ⚠️ 핵심 개념 (항상)
- CFD = Navier-Stokes(질량·운동량·에너지 보존) 이산화 수치해석. "색깔 예쁘면 끝"이 아니라
  **경계조건 타당성 → 메시 품질 → 수렴 판정 → 물리 검증(실험/문헌 대비)** 순으로 확인.
- 목적(열관리·유동저항·공력가열)에 따라 필요 모델·정밀도가 완전히 달라짐 — 목적 먼저 명확히.
- 실행은 [[fluent]] 저널(.jou)로 자동화.

## 열전달 3형태 (전도·대류·복사)
- **전도**: 고체 내부, 푸리에 `q=-k∇T`. 방열판 설계 핵심(k값이 결과 좌우).
- **대류**: 유체-고체 경계. 자연대류(부력)/강제대류(팬)를 구분해 모델(중력항 포함 여부) 설정.
- **복사**: 고온(수백°C↑, 공력가열)·진공/저유속서 지배적. 저온 전자냉각은 보통 무시 가능하나 큰 온도차/우주환경은 필수. 복사 미고려가 전자냉각 오차 주원인.
- 방산 다수는 **연성 열전달(CHT)**: 고체(전도)+유체(대류) 동시 해석해 계면 온도·열유속 자동 매칭. 전자장비 함체·방열판 표준.

## 난류 모델 (k-epsilon, k-omega SST — 언제 뭘)
- **Realizable k-epsilon**: 벽에서 먼 자유전단·내부유동(덕트·챔버)에 비용 대비 정확·안정. 벽함수로 메시 성겨도 됨. 복잡한 박리 없을 때.
- **k-omega SST**: 경계층·박리·역압력구배 중요한 외부 공력(유도탄 외형·노즐·터보기계). 근벽 y+≈1 요구 시 표준.
- **판단**: 박리·역압력구배·정밀 경계층 필요 → SST. 내부유동/메시 제한 → Realizable k-epsilon(+벽함수).
- 고속 공력가열은 SST + 천이(transition) 모델 병행. ⚠️ 극초음속 화학반응·비평형은 Fluent 기본만으론 부족 — 별도 검토.

## 경계조건 설정 원칙
- 입구: velocity-inlet(유동 알 때) 또는 mass-flow-inlet(압력 시스템 연계).
- 출구: pressure-outlet 기본. 역류 시 온도/난류 경계값 지정해야 발산 방지.
- 벽: 단열/지정 열유속/지정 온도/대류경계(h). 발열원은 heat flux·발열량(W), 외피는 대류경계.
- CHT 시 고체-유체 접촉면은 "coupled"로 자동, 별도 h 불필요.
- 대칭/주기 경계는 계산량 절감이나 받음각(AOA) 비대칭 유동엔 사용 불가.

## 메시·수렴·검증 (y+, 잔차, 격자독립성)
- **y+ 가이드** [ANSYS UG 12.3.1, LEAP]:
  - 벽함수(Standard/Non-eq): 30 < y+ < 300 · Scalable: y+ > 11.225
  - Enhanced Wall Treatment / SST / Spalart-Allmaras 근벽 직접: y+ ≈ 1 (최대 5)
  - 박리·천이·정밀 압력강하는 y+≈1 필수.
- **잔차 판정**(실무 관행, 출처 편차): 1e-4 느슨 · 1e-5 양호 · 1e-6 엄격. 잔차만으론 불충분 —
  **관심 지점 모니터값 불변 + 열/질량 불균형이 최소 경계유량의 1% 미만** 함께 확인 [ANSYS 26.13.1].
- **격자독립성**: 2~3단계 세분화(1×/2×/4×)하며 핵심결과(최대온도·압력강하) 수렴 확인.
- 방산 팁: 보고서/설계검토용은 **격자독립성+잔차+모니터값 3중 확인**한 것만 제출.

## 실무 검토 (전자장비 열관리, 유도탄 공력가열)
- **전자장비 함체**: CHT 강제/자연대류 표준. 오차 주원인 4가지 = 난류모델·복사 미고려·근벽 메시·수렴 부족. SST+Enhanced Wall Treatment가 방열판에 널리 쓰임.
- **유도탄 공력가열**: 정체점 열유속 최대. Fay-Riddell 상관식 vs CFD가 ±15% 이내 일치 사례(NASA TFAWS). 층류→난류 천이 시 가열률 급증 — 천이 위치 예측이 열방호 설계 핵심 불확실성.
- 핀-동체 간극 등 충격파-경계층 간섭으로 국부 열유속 급증 — 축대칭 근사로 놓칠 방산 리스크.
- ⚠️ 극초음속·화학반응(고온 해리)은 Fluent 표준 범위 밖일 수 있음 — 전문 솔버/상관식 별도.

## 실행 ([[fluent]] 저널과 연계)
- 실제 실행은 [[fluent]] 노트의 저널 작성 → 배치(`fluent 3ddp -g -i case.jou`). 이 노트는 모델·경계조건·수렴판정 결정 근거.
