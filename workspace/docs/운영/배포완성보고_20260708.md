## 2. 승인 모드 (사용자 결정 반영)

- **command_guard 한국어 마커 완화: 유지.** ALLOW가 아니라 ASK로만 내리고 위험 패턴 차단력은 그대로라 안전마진 유지.
- **"완전 무승인" 신규 모드: 추가 안 함.** 기존 **FULL(완전 오토)** 이 바로 그 개념 — 종류별 최초 1회만 승인하고 이후 세션 내내 무프롬프트. 어떤 모드에서도 `command-guard`의 위험 명령 하드 차단은 유지(불변 규칙).

## 4. 설치 흐름(사용자 관점)

1. 배포 폴더를 회사 PC로 옮김 → `INSTALL_OFFLINE_LIG_OPENCODE.bat` 더블클릭.
   - payload 해시 검증 → 기존 설치 백업 → workspace 미러 복사 → wheelhouse 설치 트리 복사 →
     USERDATA 시크릿 비파괴 시드 → 번들 wheel 오프라인 pip 설치 → 검증 bat 생성.
2. `%USERPROFILE%\OpenCodeLIG\workspace\RUN_OPENCODE_LIG.bat` 실행 → 무설정 게이트웨이 연결.
3. 첫 pip 일부 실패 시 `launch\install-tools.bat`로 재시도(이제 설치 트리 wheelhouse로 정상 동작).
