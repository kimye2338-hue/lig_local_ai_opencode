### 규칙 준수 확인
- **안전 가드**: 완화 없이 차단만 추가하는 방향(정상 read-only 명령 오탐 없음 확인).
- **오프라인 전제**: 신규 외부 네트워크 호출 없음(browser_cdp는 로컬 CDP 전용).
- **SolidWorks 2022 API**: 문서타입 상수·매크로 시그니처 정합.
- **lig_runtime emergency_local 승인 조건**: 정상 텍스트 응답을 거부하지 않음(평문 응답은 `parse_status="none"`이라 통과) — 회귀 아님.

### 수정 완료 (High→처리)
- **`command_guard.py` — `--output` 따옴표 우회 (수정함)**
  - 문제: 새로 추가된 `_safe_prefix_writes_output`가 `git diff --output=x`는 막지만 `git diff "--output=x"`처럼 따옴표로 감싸면 정규식 선행조건에 안 걸려 파일 쓰기가 자동 허용(safe_prefix)으로 통과.
  - 수정: 선행 문자 클래스와 옵션 뒤 구분자에 따옴표를 포함하도록 정규식 확장.
    - 이전: `r"(?:^|\s)(?:--output(?:=|\s+)|-o(?:\s+|=))\S+"`
    - 이후: `r"""(?:^|[\s"'])(?:--output(?:[=\s]|["'])|-o(?:[\s=]|["']))\S"""`
  - 검증: `git diff "--output=x"` → `safe_prefix=False`, 결정 `ask`(자동 허용 안 됨). 정상 명령(`git diff --stat`, `--output-indicator-new=+`, `--oneline`) 오탐 없음. approval 테스트 23개 통과.

### 잔여 경미 (수정 불요, 참고)
- `input_ingest.py` xlsx markitdown 폴백의 `convert_file`이 예외를 삼지 않아 이론상 raise 가능 — **기존 코드도 동일 패턴이라 새 회귀 아님**.
- `verifier.py`의 `ensure_memory()`가 `memory.jsonl`을 즉시 생성 → 바로 아래 존재검사가 자기충족적. 버그 아님, 검증 의미만 약해짐.
- `command_guard.py`의 `-o` 분기는 `git diff/log`엔 `-o` 옵션이 없어 사문(死文). 무해.

### 개선 권장

**[높음] A. 오프라인 복구 경로 단절 — 설치본 `tools/`가 비어 `install-tools.bat`이 깨짐**
- 원인: 설치기(`INSTALL_OFFLINE_LIG_OPENCODE.bat.txt:72`)가 `robocopy ... /XD "%WS%\tools"`로 `tools/`를 통째로 제외 → 설치된 워크스페이스의 `tools/`는 `:79`의 `mkdir`로 **빈 폴더**만 남는다.
- 영향: 최초 설치 중 번들 pip(`:138`)는 **소스 패키지**의 wheelhouse에서 설치하므로 정상. 하지만 그게 일부 실패하면 설치기가 안내하는 재시도 경로 `launch\install-tools.bat`이 `%CD%\tools\wheelhouse`(=설치본, 빈 폴더)를 보고 `:18`에서 "wheelhouse 가 없습니다"로 종료 → **원본 패키지 USB 없이는 선택기능(Office/markitdown/OCR) 오프라인 복구 불가.**
- 의도 충돌: `/XD tools` 주석은 "사용자 반입 바이너리를 재설치 간 보존"이 목적 — 재설치 보호는 맞으나 **최초 설치에서 패키지 동봉 `tools/`(wheelhouse·Obsidian setup)를 시드하지 않는** 점을 놓침.
- 권장 수정(택1):
  1. 설치기에서 **최초 설치(대상 tools가 비었을 때)에 한해** 패키지 `tools\wheelhouse`를 설치 트리로 개별 robocopy 시드. (재설치 시 사용자 반입분 보존은 유지)
  2. 또는 `install-tools.bat`이 설치본에 wheelhouse가 없으면 원본 패키지 경로로 폴백.
- 최소 패치안(설치기, `mkdir "%WS%\tools"` 다음 줄에 추가):
  ```bat
  rem 최초 설치 시에만 번들 wheelhouse 를 설치 트리에 시드(오프라인 재설치 복구용).
  if not exist "%WS%\tools\wheelhouse" (
    robocopy "%~dp0workspace\tools\wheelhouse" "%WS%\tools\wheelhouse" /E /NFL /NDL /NJH /NJS /NP >nul
  )
  ```

**[중] B. certutil 부재 시 무결성 검증 폴백 없음**
- 설치기 `:37`에서 certutil 실패 시 "Failed to calculate checksum"으로 즉시 중단. certutil은 통상 존재하나, 무결성 검증이 유일한 게이트이므로 `powershell Get-FileHash` 폴백 추가 권장.

**[경미] C. `deps`의 Obsidian 탐지 경로 불일치(표시만)**
- `cmd_deps`가 `~/OpenCodeLIG/tools/Obsidian/Obsidian.exe`를 찾는데 패키지는 `tools/Obsidian/…-setup.exe`(설치 전 인스톨러) → deps에서 항상 "미검출" 표기. 기능 영향 없음, 표시 정확도만.

**[경미] D. `doctor`의 `opencode --version`은 PATH 의존**
- bin 미등록/신규 터미널 전이면 에러 문자열만 기록(비차단).
