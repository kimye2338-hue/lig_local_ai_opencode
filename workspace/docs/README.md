# OpenCodeLIG 문서 — 여기 하나로 전부 파악

사내 오프라인 한국어 AI 업무비서. **폴더 이름만 보고 원하는 문서를 찾으면 됩니다.**

## 📁 폴더 지도

| 폴더 | 무엇이 있나 | 언제 보나 |
|---|---|---|
| **[사용법/](사용법/)** | 설치·사용 가이드·문제해결 | 설치하고 쓰기 시작할 때 |
| **[기능/](기능/)** | 기능별 상세(문서변환·Office·OCR·Obsidian·데스크톱펫·기억) | 특정 기능을 깊게 알고 싶을 때 |
| **[설계/](설계/)** | 아키텍처·하네스 원칙·저장소 지도·OpenCode 통합·설계 스펙·결정기록 | 개발·확장·이어작업 할 때 |
| **[운영/](운영/)** | 반입 툴·외부도구 판정·변경이력·런타임 교훈·인수인계 | 배포·운영·트러블슈팅 할 때 |
| **[archive/](archive/)** | 통합 전 옛 문서(보존용, 최신 아님) | 과거 맥락이 필요할 때만 |

## 🚀 처음이라면

1. **설치·사용**: [사용법/GUIDE.md](사용법/GUIDE.md) 하나면 설치·사용·문제해결까지 됩니다.
2. **전체 그림**: [설계/MASTER_PLAN.md](설계/MASTER_PLAN.md) (제품 비전·전략) + [설계/REPOSITORY_MAP.md](설계/REPOSITORY_MAP.md) (저장소 구조).
3. **이어서 개발**: 루트 `CLAUDE.md`(세션 진입점) → 이 문서 → [설계/specs/](설계/specs/)의 최신 계획.

## 🧩 기능 한눈에 (자연어로 시키면 자동 선택)

산출물 생성, 진짜 Office 파일, 데이터 HTML 리포트, 정형문서, 문서 읽기, 웹 CDP + 화면 OCR,
앱 자동화(어댑터 — 대화형에서 직접 호출), 반복작업 record&replay, 활동 타임라인, 무한대기 감시,
기억·위키(자동 적재 → Obsidian 자동 연동 → 하루 2회 자율 정리). 작업 유형별로 공식 API·디자인·
한국 비즈니스·절차 지식이 자동 주입됩니다. 상세는 위 폴더 참조.

## 📐 문서 관리 원칙

- **단일 진입**: 이 `docs/README.md` 가 유일한 목차. 새 문서는 위 5개 폴더 중 하나에 넣고 여기 표에 한 줄 추가.
- **단일 진실원**: 능력·도구·상태의 사실은 코드(registry)가 기준. 문서는 설명·링크만 하고 중복 서술하지 않음.
- **무손실**: 통합·폐기 대신 `archive/` 로 보존.
