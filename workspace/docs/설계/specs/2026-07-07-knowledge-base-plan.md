# 세계 최고 조수 만들기 — 대규모 팩트체크 지식베이스 설계

작성 2026-07-07 (Fable 5 직접 설계 — advisor 도구 미가용으로 Fable이 아키텍처 담당).
상태: **계획(사용자 확인 대기)**. 확인 후 수집·구축, 패키지는 마지막 일괄.

목표: 사용자가 "이거 만들어줘/이 작업 해줘"만 하면, 정확한 근거로 즉시 구현하는 조수.
소프트웨어 + 공학 도메인 + 회사생활 소프트스킬을 팩트체크된 지식으로 총망라, Obsidian에
유기적으로 구성, 약한 모델(EXAONE/Qwen)이 정확히 꺼내 쓰게.

---

## 1. 두 지식 시스템을 분리한다 (핵심 결정)

| | 개인 기억 (있음) | 레퍼런스 지식베이스 (신설) |
|---|---|---|
| 위치 | `USERDATA/memory` (불가침) | `agent_ops/knowledge/` (코드측, 패키지 동봉) |
| 성격 | 이 사용자·작업에서 배운 것, 자동 생성 | 큐레이션된 사실(소프트웨어·도메인·스킬) |
| 수명 | 자동 삭제 금지, 사용으로 증식 | 버전관리·팩트체크·갱신 가능 |
| Obsidian | wiki_vault 자동 시드(완료) | **같은 방식으로 knowledge/ 도 vault화** |

둘 다 Obsidian 그래프로 열리고 둘 다 검색에 주입되지만 **물리적으로 분리** — 개인 기억을
레퍼런스가 오염시키지 않고, 레퍼런스는 갱신해도 개인 기억이 안전.

## 2. 레퍼런스 KB 구조 (`agent_ops/knowledge/`)

```
knowledge/
  _index.md            마스터 색인(전 도메인 MOC 링크)
  _moc/                Maps of Content — 도메인 허브(하위주제 1줄요약+[[링크]])
  apis/                소프트웨어 API (기존 확장 + python/cmd/windows 추가)
  domains/             공학 도메인 노트 (아래 3-B 목록)
  standards/           규격 (MIL-STD-810H, 금속/나사 규격 등)
  skills/              소프트스킬 (협상·커리어·ADHD·성과·문서·논문) — 기존 process skills와 구분
  .obsidian/           vault 설정(자동 시드 — wiki_vault 재사용)
```

각 노트 = 하나의 주제. 프론트매터로 팩트체크 상태를 싣는다(§4).

## 3. 커버리지 (단계적, 사용자 요청 전부 포함)

**A. 소프트웨어**: CAD(AutoCAD 2019), ANSYS(Fluent 2024R1 + Mechanical 구조/진동), SolidWorks
2022, MATLAB R2024a, Python(엔지니어링: numpy/scipy/pandas/matplotlib), CMD/PowerShell, Windows.
**B. 공학 도메인**: 유도탄 설계, 구조해석, 진동해석, 열유체해석(CFD), 기구설계, 나사·체결,
금속규격, 공력, 전자공학, 하드웨어 설계, 케이블 설계, MIL-STD-810H, 치구설계 + (제안 추가:
공차·GD&T, 재료역학, 피로/파괴, 유한요소법 기초, EMC/EMI, 신뢰성, 형상관리).
**C. 소프트스킬**: 협상의 기술, 커리어 관리, ADHD 친화 일정·집중 관리, 성과관리, 문서작성,
논문작성법 + (제안 추가: 회의 진행, 이메일/보고 커뮤니케이션, 시간관리).

## 4. 팩트체크 보증 구조 (Fable 설계 — 정확성이 생명)

각 노트 프론트매터:
```yaml
---
title: 진동해석 기초
domain: 진동해석
sources: [출처 URL/문서 ...]     # 출처 없는 주장 금지
verified: true | draft            # 검증 통과 여부
confidence: high | medium | low
version: (해당 시 소프트웨어/규격 버전)
reviewed: 2026-07-07
---
```
**수집→검증 2단 파이프라인**:
1. **수집**(Haiku/Sonnet, 저비용): 주제별로 1차 소스에서 사실+출처를 모은다. **주장마다 출처
   필수**, SEO 저품질 사이트 배제(공식/표준/교과서/논문 우선).
2. **적대적 검증**(별도 저비용 에이전트): 각 주장이 출처와 일치하는지 교차검증 → 불일치·무출처는
   confidence 강등 또는 `draft`. 규격/수치/버전 종속 사실은 특히 엄격.
- 주입 시 `verified:true`만 정상 주입, `draft`는 "미검증" 꼬리표를 달거나 고위험 작업엔 제외.
- 수치·규격은 "출처에 이렇게 명시"를 병기해 모델이 지어내지 않게.

## 5. 검색(주입) 아키텍처 — 약한 모델이 정확히 꺼내오게

현 주입은 키워드+relevance 발췌(소수 파일용). 수백 노트·25+ 도메인엔 2단 검색:
1. **도메인 라우팅**: 작업 프롬프트 → 관련 도메인 감지(키워드/별칭). (api_reference의
   detect_software를 도메인으로 일반화.)
2. **MOC 주입**: 해당 도메인 허브(하위주제 1줄요약 목록)를 먼저 주입 → 모델이 "무엇이 있는지"
   지도(map)를 본다. (Aider repo-map 패턴.)
3. **relevance 발췌**(완료): 선택된 노트에서 작업 관련 섹션만 예산 내 주입.
4. **importance/신뢰도 가중**(완료된 기억 랭킹과 동일 축): verified·high confidence 우선.
= map(MOC) → detail(relevant excerpt), 예산 관리. 약한 모델 컨텍스트 낭비 최소.

## 6. 유기적 연결 (Obsidian + 모델 양쪽)

- **MOC 허브**: 도메인마다 `_moc/<도메인>.md` 가 하위주제를 [[링크]]로 묶음(사람 네비 + 모델 인덱스).
- **교차 도메인 링크**: 예 진동해석 ↔ MIL-STD-810H ↔ 유도탄설계 ↔ 치구설계 를 [[링크]]로 연결 →
  Obsidian 그래프에 실제 지식 지도가 뜨고, 한 작업이 여러 도메인을 자연히 끌어온다.
- **소프트웨어↔도메인 연결**: 예 구조해석 도메인 노트가 [[ANSYS Mechanical]] API 노트를 링크 →
  "구조해석 해줘"가 이론+실행(API)을 함께 주입.

## 7. 수집 파이프라인 (토큰 최적화 — Haiku/Sonnet)

- **Fable(나)**: 스키마·MOC 구조·검증 루브릭·연결(링크) 설계 + 최종 배선. (고난도)
- **Haiku/Sonnet 서브에이전트**: 도메인·주제별 벌크 수집(저비용). 워크플로로 병렬 파이프라인
  (수집→검증→구조화) 구성. 각 산출은 §4 프론트매터 규격으로 바로 ingest 가능하게.
- 도메인 단위로: 수집 → 적대적 검증 → knowledge/에 저장 → MOC·교차링크 배선 → 검색 스모크.

## 8. "말만 하면 바로 구현" 동작 (완성 후)

"유도탄 치구 진동 시험 지그 설계 검토해줘" →
도메인 라우팅(진동해석·치구설계·MIL-STD-810H) + 소프트웨어(ANSYS/SolidWorks) 감지 →
MOC로 관련 주제 지도 + verified 이론·규격·API 레시피 주입 → 모델이 근거로 즉시 산출 →
결과가 개인 기억에 적재 → 다음엔 더 빨라짐(복리).

## 9. 실행 단계 (승인 후, 패키지는 최종 일괄)

0. 아키텍처 구축(Fable): knowledge/ 구조 + 도메인 라우팅 주입기 + MOC 로더 + knowledge vault 시드.
1. 소프트웨어 KB(진행 중 리서치 3건 반영 + Python/CMD/Windows).
2. 공학 도메인 KB(수집→검증→연결).
3. 소프트스킬 KB.
4. 전 회귀 green → 패키지 1회 재빌드(지식 포함 — 용량 큼, 분할 조각 늘어남).

## 10. 사용자 확인 필요

- 커버리지 §3 목록(특히 "제안 추가" 항목) 포함 여부.
- 지식 저장 위치: 코드측 `knowledge/`(패키지 동봉·버전관리) + Obsidian vault화 방향 OK?
- 팩트체크 엄격도: verified/draft 2단 + confidence로 충분한지, 더 엄격히 할지.
- 규모/우선순위: 어느 도메인부터(내 실무 직결 순서).
